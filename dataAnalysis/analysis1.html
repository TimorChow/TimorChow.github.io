1 Data understanding¶

DataSet: online_retail_II¶

Description:¶

This Online Retail II data set contains all the transactions occurring for a UK-based and registered, non-store online retail between 01/12/2009 and 09/12/2011.The company mainly sells unique all-occasion gift-ware. Many customers of the company are wholesalers. Download DataSet

Attribute Information:¶

InvoiceNo: Invoice number. Nominal. A 6-digit integral number uniquely assigned to each transaction. If this code starts with the letter 'c', it indicates a cancellation.

StockCode: Product (item) code. Nominal. A 5-digit integral number uniquely assigned to each distinct product. Description: Product (item) name. Nominal.

Quantity: The quantities of each product (item) per transaction. Numeric.

InvoiceDate: Invice date and time. Numeric. The day and time when a transaction was generated.

UnitPrice: Unit price. Numeric. Product price per unit in sterling (Â£).

CustomerID: Customer number. Nominal. A 5-digit integral number uniquely assigned to each customer.

Country: Country name. Nominal. The name of the country where a customer resides.

2.4 RFM descretization¶

In [11]:
"""
Note: 1-5 is weight. F and M, higher values are better, which has a higher weight. Recency is opposite, smaller value has higher weight.

注意：1-5来对应离散后的值，Frequency和Monetary数值越大权重越大，
Rececny相反，Recency越小权重越大。
"""
# make it a dict, easy to use.
quantiles = rfmTable.quantile(q=[0.2, 0.4, 0.6, 0.8])
quantiles = quantiles.to_dict()

def r_class(x, category, df):
    """
    x:  data before descretization. 离散化之前的数据
    category: R, F, M
    df: dataframe of quantiles      用于分段的分位数
    """
    if x<=df[category][0.2]:
        return 5
    elif x<=df[category][0.4]:
        return 4
    elif x<=df[category][0.6]:
        return 3
    elif x<=df[category][0.8]:
        return 2
    else:
        return 1

def fm_class(x, category, df):
    if x<=df[category][0.2]:
        return 1
    elif x<=df[category][0.4]:
        return 2
    elif x<=df[category][0.6]:
        return 3
    elif x<=df[category][0.8]:
        return 4
    else:
        return 5

"""
RFM discretization
"""
rfmSeg = rfmTable
rfmSeg['R_Seg'] = rfmSeg['Recency'].apply(r_class, args=('Recency', quantiles))
rfmSeg['F_Seg'] = rfmSeg['Frequency'].apply(fm_class, args=('Frequency', quantiles))
rfmSeg['M_Seg'] = rfmSeg['Monetary'].apply(fm_class, args=('Monetary', quantiles))

2.5 Add Response¶

Define what is valuable customer?

Make response data: Response is an indicator that shows if a customer has membership.

Add response for customer. 1 means this customer is valuable, 0 means not.

rule 1: next 4 month, this customer come again.

rule 2: according to whole purchase life, a customer who comes less than every 30 days on average. (day can be change)

为训练集添加response, 1表示认为客户是会员

规则1: 接下来的4个月 重复出现了该用户1次, 则response值为1

规则2: 首次购买和最后一次购买期间, 平均购买时间间隔短的, 视为会员,如平均30天购买一次

def gainTable(y_true_int, y_pred_prob):
    """
    y_true_int: 
    y_pred_prob: predict probability
    return: DataFrame
    """
    data = pd.DataFrame({'y_true': y_true_int, 'prob': y_pred_prob})
    data['prob'] = 1 - data['prob']
    data['percentile_gp'] = pd.qcut(data['prob'], q=10, labels=range(10))
    deciles = data.groupby('percentile_gp', sort=True)
    def total_count(x): return len(x)
    def pos_count(x): return np.sum(x)
    
    # positive rate
    def pos_rate(x): return np.sum(x)/float(len(x))
    out = deciles['y_true'].agg([total_count, pos_count, pos_rate])
    
    # negitive
    out['neg_count'] = out['total_count'] - out['pos_count']
    
    # positive total sum
    out['pos_cumsum'] = out['pos_count'].cumsum()
    total_sum = out['pos_cumsum'][-1]
    
    # positive cover rate
    out['pos_cover_rate'] = out['pos_cumsum'] / float(total_sum)
    out['percentile'] = (out.index.astype(int) + 1) / 10.0
    return out[['percentile', 'total_count', 'pos_count', 'neg_count', 'pos_rate', 'pos_cover_rate']]
